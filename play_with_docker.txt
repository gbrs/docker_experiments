Здравствуйте, курс вполне уверенно проходился в виртуальной среде play-with-docker.
Из личных ощущений:
Из плюсов - нет привязки к характеристикам компьютера (мой процессор не поддерживал инструкции необходимые для запуска ClickHouse);
при запуске web-сервиса отображает кнопку со ссылкой на него в "админской панели".
Из минусов - ограничение на объем памяти (4ГБ памяти диска иногда не хватало).

На данный момент я еще не завершил курс по docker (остановился на 7 уроке), но судя по дальнейшим темам, проблем которые бы мешали не видно. Весь инструментарий docker доступен "из коробки".

Если глобально, то алгоритм следующий:
Регистрируемся в hub.docker.com
Настраиваем ssh-клиент (в моем случае, putty под Win7)
2.1. Устанавливаем Putty
2.2 Генерируем приватный ключ и устанавливаем пароль (как я понял можно взять любой имеющийся и вообще не использовать пароль, но, это будет не педагогично)
2.2.1 Используем встроенную утилиту PuttyGen (Generate, Тип ключа - RSA), возим мышкой сохраняем получившийся приватный ключ в файл
2.2.2 Указываем путь к приватному ключу в настройках
[Connection->SSH->Auth->Credentials (поле "Private key ...")]
2.2.3 Сохраняем конфигурацию в разделе Session, чтобы не указывать ключ каждый раз при запуске Putty.
Подключаемся к play-with-docker
3.1. Входим в "админку" по URL labs.play-with-docker.com и создаем виртуальную машину.
3.2 Копируем новый адрес ВМ из поля SSH и подставляем его в Putty (Session поле HostName (or IP adress) из раздела настроек (предварительно нажимаем Load, чтобы загрузить профиль с нашим приватным ключом).
3.3 Запускаем Putty и в открывшемся терминале вбиваем пароль приватного ключа.
Profit!
