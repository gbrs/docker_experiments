Можно устанавливать и через Windows/Docker Desktop, и через Ubuntu/Docker.

docker volume create superset_db
docker run -d --rm -p 80:8088 -v superset_db:/app/superset_home -e "SUPERSET_SECRET_KEY=SprSt1504" --name superset apache/superset
docker exec -it superset superset fab create-admin --username admin --firstname Superset --lastname Admin --email admin@superset.com --password admin
docker exec -it superset superset db upgrade
docker exec -it superset superset init
docker exec -it superset superset load_examples

Остановить:
docker stop superset

В следующий раз перезапускать получается одной командой:
docker run -d --rm -p 80:8088 -v superset_db:/app/superset_home -e "SUPERSET_SECRET_KEY=SprSt1504" --name superset apache/superset

Резюме и правильный алгоритм
Первый запуск:
docker volume create superset_db (создаем том)
docker run -d -p 8080:8088 -v superset_db:/app/superset_home --name my_superset apache/superset:latest (запускаем контейнер с томом)
docker exec ... superset db upgrade (инициализируем БД один раз)
docker exec ... superset init (создаем учетную запись admin один раз)

После перезагрузки/остановки Docker:
Открываем Docker Desktop (ждем, пока он запустится).
В консоли пишем: docker start superset
Идем в браузере на http://localhost:8080 — все ваши дашборды и настройки на месте.

Если что-то пошло не так и старый контейнер "сломался":
docker stop superset (останавливаем старый)
docker rm superset (удаляем старый контейнер)
Запускаем команду из пункта 1.2 (с тем же томом superset_db) — данные восстановятся.
Таким образом, образ (apache/superset:latest) вы качаете только один раз. А ваши данные сохраняются благодаря тому
(superset_db), который переживает перезапуски и удаления контейнеров.
